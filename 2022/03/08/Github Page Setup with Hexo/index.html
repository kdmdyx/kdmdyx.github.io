Github Page Setup with Hexo
Mar 8, 2022

Rant

After wrestling with bash for two days, this blog is finally up and running! For my first post, I am going to document this process to hopefully pave the road for blogging greenhorns in future.

Background

This guide is meant to provide a broad picture of the components and procedures requried to build a Github Page with Hexo from scratch. With a bonus of minor issues I ran into and how to resolve them.

The core components at play are:

  • Github Page
    • A special Github repo that provides a free access point to your webpage on the internet.
  • Node.js
    • The backbone framework required by Hexo.
  • Hexo
    • A Node.js based package that supplies the tools to construct and manage your blog.

The procedures involves 3 major steps:

  • Setup Local Environment
  • Build and Test Locally
  • Deploy from Local to Github Page

This guide assumes its reader has some exposure with bash shell/command line interface and Github. The original environment used here is Windows 11. Other OS should also be able to follow along, since most work is done through UNIX shell commands.

Part I: Setup Local Environment

Install Windows Subsystem for Linux (WSL)

For starters we are going to need Windows Subsystem for Linux(WSL). Non-Windows user can skip this part.

  • Open Windows command line and run wsl --install in the prompt
  • If you do not have WSL installed already, a separate prompt should show up for a one-time setup.
  • Follow instructions on the new prompt window, pick username and password.
    • Linux username has a strict format, try to use short phrase that:
      • Starts with lower case letter
      • Uses lower case letter and number combination.
  • After finishing setup, reboot your system and try to launch bash shell by searching for bash in start menu.
  • If you have difficulties, check https://docs.microsoft.com/en-us/windows/wsl/install for official installation guide.

Perform the following steps in bash or equivalent UNIX shell of your system.

Install Git

Git the version management tool, required by Hexo to pull from its repo.

  • Simply run sudo apt-get git in your shell
  • Check out original guide for details: https://git-scm.com/downloads

Install Node.js and Hexo

Node.js is a web framework required by Hexo.

  • Get version manager for Node.js by running npm install -g n
  • Get the latest official release by running n install latest
  • Check Node.js version by running node -v for confirmation
  • Get Hexo by running npm install -g hexo-cli
  • Check Hexo version by running hexo -v for confirmation
  • Check out original guide for details:
    • https://www.npmjs.com/package/n for Node.js
    • https://hexo.io/docs/index.html for Hexo

Part II: Build and Test Locally

Create Local Site with Hexo

In this step we generate a bare-bone site locally using Hexo, which can be deployed online later.

  • In bash, navigate to a folder of your choice for your new web page files.
    • For windows users, you may also navigate in file explorer, then type bash in the address bar near top then hit “Enter” to launch bash directly there.
    • BONUS: Edit the registry to add bash to the Right-Click drop down menu for easy access anywhere: TODO: MAKE POST.
  • Run command hexo init <folderName> to generate necessary files in a new folder with .
    • Alternatively, hexo init to generate files in current folder. The folder must be empty.
  • Navigate inside the new folder and run command npm install.
  • NOTE: If you run into errors here which states some function does not exist, make sure your Node.js version is updated and is compatible with your Hexo version.
  • OPTIONAL: Configure your new site as you see fit using Hexo documentation.
  • Run hexo generate to generate static files.
  • Run hexo server to launch the website locally to preview in browser. Press Ctrl+C key to stop.
  • In a browser, navigate to http://localhost:4000 to view your page.

Congratulations, you now have a website locally. Now we just need to setup the remote end for deployment.

Part III: Deploy from Local to Github Page

Setup Github Page and Authentication

Github Page is a free service for Github users, which provides an access point to your website on the internet.

  • Log in to Github and create a new repo with name of this format: <github_username>.github.io
  • Check out official setup guide for details: https://pages.github.com/
  • BONUS: For first time git users, you will need to generate an SSH key for authentication with Github during deployment. In plain words, you need to create a certificate on your system, and show Github “this is me” using the same certificate both prior and during deployment.
    • Official guide for key generation
    • Official guide to add generated key to your Github account

Deployment

Hexo has a simple one-command deployment support for Github Page.

  • Run command npm install hexo-deployer-git --save to install the once command deployer helper
  • Edit _config.yml in hexo generated folder
    • Edit or append a deploy section near the end of the file similar to this format:
    • Feel free to deploy to different branch
  • Mored advanced options for deployment in official deployer guide
  • Clean up directories with hexo clean
  • Deploy to Github Page with hexo deploy

Voila! Your Github Page is live. Go to .github.io in a browser to view. It might take a minute for the page to update.

What Now?

  • The official Hexo guide has a lot more options to customize your page.
  • The essential workflow with Hexo looks like:
    • Edit some files.
    • hexo generate to generate static files after editing.
    • hexo server to launch a local version for quick preview.
    • hexo clean for house keeping before deployment.
    • hexo deploy to push changes to live page.

Enjoy!
